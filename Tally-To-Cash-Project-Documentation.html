üîê Google OAuth Setup Guide (Complete Steps)

Problem: "Unable to access backend" - Google OAuth redirect_uri_mismatch or authentication errors.

Solution: Follow these exact steps to fix Google OAuth authentication between frontend and backend.

Step 1: Test Backend Health (Basic Check)

# Test if backend is running at all curl https://tally-backend-597099030754.asia-south2.run.app/api/health # Expected response: {"status":"ok","timestamp":"..."} # If this fails, backend is down or unreachable

Step 2: Find Your OAuth Client in Google Cloud Console

  1. Go to Google Cloud Console ‚Üí APIs & Services ‚Üí Credentials
  2. Look for OAuth 2.0 Client with ID: 597099030754-c6i7kp1nn4iq8dcrlhf7j7qogsso9tnv.apps.googleusercontent.com
  3. Click on it to edit
  4. If you don't see it, you need to create a new "Web application" OAuth client

Step 3: Configure OAuth Client (Exact Settings)

‚ö†Ô∏è CRITICAL: Add these EXACT entries to your OAuth client:

Authorized JavaScript origins:

https://accountancy-469917.web.app https://accountancy-469917.firebaseapp.com http://localhost:5173 http://localhost:3000

Authorized redirect URIs:

https://accountancy-469917.web.app https://accountancy-469917.web.app/login https://accountancy-469917.firebaseapp.com https://accountancy-469917.firebaseapp.com/login http://localhost:5173 http://localhost:5173/login http://localhost:3000 http://localhost:3000/login

Step 4: Add Client Secret to Backend (REQUIRED)

The backend CANNOT work without the client secret. Add it to Cloud Run:

# Get the client secret from Google Cloud Console ‚Üí Credentials ‚Üí Your OAuth Client # Then run this command (replace YOUR_SECRET_VALUE): gcloud run services update tally-backend \ --region=asia-south2 \ --update-env-vars="GOOGLE_CLIENT_SECRET=YOUR_SECRET_VALUE" # Verify it was set: gcloud run services describe tally-backend --region=asia-south2 --format="value(spec.template.spec.template.spec.containers[0].env)"

Step 5: OAuth Consent Screen (For External Users)

  1. Go to OAuth consent screen
  2. If it's "Internal" and you want external users to sign in, change to "External"
  3. Fill required fields: App name, User support email, App logo (optional)
  4. Add test users OR publish the app for public use
  5. Save and continue through all steps

Step 6: Test the Full Flow

# 1. Open browser to your frontend https://accountancy-469917.web.app # 2. Open DevTools ‚Üí Network tab # 3. Click "Sign in with Google" # 4. Check for these requests: # - accounts.google.com/o/oauth2/v2/auth (should succeed) # - YOUR_BACKEND/api/auth/google (should return 200, not 500) # 5. If you see errors, check the exact error message

Common Error Messages & Fixes

‚ùå "redirect_uri_mismatch"

Cause: Frontend origin not in OAuth client's JavaScript origins

Fix: Add exact origin (https://accountancy-469917.web.app) to Authorized JavaScript origins in Step 3

‚ùå "Failed to fetch" or "Network Error"

Cause: Backend error (usually missing GOOGLE_CLIENT_SECRET)

Fix: Complete Step 4 to add client secret to Cloud Run

‚ùå "This app is blocked"

Cause: OAuth consent screen not configured or not published

Fix: Complete Step 5 to configure consent screen

‚ùå "CORS error"

Cause: Backend CORS not allowing frontend origin

Fix: Backend already configured for your domains, but verify environment variables are correct

Step 7: Verify Everything Works

# After completing all steps above: # 1. Clear browser cache/cookies # 2. Go to https://accountancy-469917.web.app # 3. Click "Sign in with Google" # 4. You should see Google popup ‚Üí consent ‚Üí success ‚Üí logged in # Test backend access: # After logging in, check if you can access protected endpoints like: # /api/dashboard, /api/collections, etc.

How OAuth Works in This App (Technical)

  1. Frontend: User clicks "Sign in" ‚Üí Google Identity Services opens popup
  2. Google: User consents ‚Üí Google returns authorization code to frontend
  3. Frontend: Sends code to backend: POST /api/auth/google {authCode}
  4. Backend: Exchanges code for tokens using client secret ‚Üí creates session
  5. Frontend: Gets session cookie ‚Üí can access protected endpoints
